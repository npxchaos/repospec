# acme-billing

Usage-based billing service for the Acme platform. Internal finance team plus the public billing API.

Stack: TypeScript / Node 20 / NestJS / PostgreSQL. Deploy: AWS ECS via GitHub Actions. Compliance: SOC2, PCI-DSS.

## Constitution (non-negotiable)

These rules are non-negotiable. Every change, human or agent, obeys them.

1. Small steps over large rewrites. Prefer the smallest change that ships value.
2. Tests precede implementation for any code touching billing math (see `rules/testing.md`).
3. A human approves every change under `deploy/` and `migrations/`.
4. Secrets never enter the repo, the logs, or agent output (see `rules/security.md`).
5. Money is integer cents. Never floats. Never.
6. Every externally observable behavior change is recorded in `history/`, and if architectural, in `decisions/`.

Humans amend this file. Agents obey it and never edit it.

## Roles

Work moves through defined roles, each with bounded authority. Operate as exactly one role at a time.

- **architect** - owns scope, system design, and architectural decisions. May define interfaces and record decisions; may not write production code.
- **builder** - owns implementation and documentation of changes. May write source and tests; may not approve own work or deploy.
- **reviewer** - owns review of every diff before merge. May block a merge; may not modify source.
- **tester** - owns validation that a change does what it claims and breaks nothing else. May block on failing checks; may not modify source.
- **security** - owns the security posture of changes and the repo. May block a merge or release on a finding; may not modify source.
- **release** - owns deployment, tagging, and post-release sync. May deploy behind a human gate; may not write feature code.

Full definitions: `.repospec/agents/`.

## Workflow

Plan → Design → Implement → Review → Validate → Document → Release → Sync.
Steps can't be skipped; human gates (Plan, Release) can't be auto-cleared. Details: `.repospec/workflow.md`.
