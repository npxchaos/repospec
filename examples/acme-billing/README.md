# Repospec example — acme-billing

A worked example of a Repospec-governed repository: a usage-based billing service
with six roles, billing-specific rules, and a declared plugin. Copy the
`.repospec/` directory into your own repo, edit it to describe your project, and
run `repospec sync` to regenerate the tool entrypoints.

- `.repospec/` — human-authored source of truth (you edit this)
  - `project.yaml`, `constitution.md`, `architecture.md`, `workflow.md`
  - `agents/` — the roles (architect, builder, reviewer, tester, security, release)
  - `rules/` — enforceable rules (security, testing, style)
- `AGENTS.md`, `CLAUDE.md` — generated by `repospec generate`/`sync`; carry a
  managed header — don't hand-edit (regenerate instead).

`project.yaml` also **declares** a plugin (`acme-compliance-pack`) to show the
plugin surface. A plugin executes only when approved (`repospec plugins approve`)
and run with `repospec generate --plugins` — see
[`docs/commands.md`](../../docs/commands.md).
